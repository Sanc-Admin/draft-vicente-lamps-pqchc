# QuantaScript QS-dev Integration — draft-vicente-lamps-pqchc

## Overview

This document specifies how the PQCHC draft repository integrates with the
**QuantaScript QS-dev** compiler and verification pipeline.

The integration adds two capabilities to QS-dev:
1. **Verification hook** — after a successful build/validate cycle, signal QS-dev that the IETF draft artifact passed structural verification.
2. **Functionality extension** — expose PQCHC ASN.1 types as first-class QuantaScript types for use in PKI automation scripts.

## Environment Variables

| Variable | Required | Description |
|---|---|---|
| `QS_DEV_ENDPOINT` | Optional | Base URL of QS-dev API (e.g. `http://localhost:8080`). If set, `verify.sh` posts a verification signal. |
| `QS_DEV_TOKEN` | Optional | Bearer token for authenticated QS-dev API calls |

## Verification Hook

After `tools/verify.sh` completes successfully, it posts:

```http
POST $QS_DEV_ENDPOINT/hooks/ietf-draft-verified
Content-Type: application/json

{
  "draft": "draft-vicente-lamps-pqchc-00",
  "status": "verified",
  "timestamp": "<ISO-8601>"
}
```

QS-dev can consume this event to:
- Update a draft status board
- Trigger dependent automation (e.g., certificate test harness)
- Emit a QuantaScript event for downstream policy evaluation

## QuantaScript Type Extensions

Add the following to your QS-dev type registry to expose PQCHC types:

```quantascript
// PQCHC forward commitment type
type PQCHCommitment {
  commitmentValid: bool
  committedAlgorithm: AlgorithmIdentifier
  futureKeyCommitment?: FutureKeyCommitment
  commitmentNotAfter: datetime
  policyURI?: string
}

type FutureKeyCommitment {
  hashAlgorithm: AlgorithmIdentifier
  spkiHash: bytes
  keyAlgorithmHint?: AlgorithmIdentifier
}

// Verify a PQCHC commitment at renewal time
fn verify_pqchc_commitment(
  commitment: PQCHCommitment,
  successor_spki: bytes
) -> CommitmentResult {
  let candidate_hash = sha384(successor_spki)
  if candidate_hash == commitment.futureKeyCommitment.spkiHash {
    return CommitmentResult::Fulfilled
  }
  return CommitmentResult::Violated
}
```

## Adding Functionality to QS-dev

To register these types with QS-dev at startup:

```bash
curl -sSf -X POST "$QS_DEV_ENDPOINT/registry/types" \
  -H "Authorization: Bearer $QS_DEV_TOKEN" \
  -H 'Content-Type: application/json' \
  -d @tools/qs_dev_type_registry.json
```

See `qs_dev_type_registry.json` (generated by `verify.sh --export-types`) for
the full registry payload.

## Security Note

The QS-dev integration exposes only the **public wire format** (ASN.1 extension
structure). No Sanctum SecOps internal PKI engine, CA automation, or QuantaScript
runtime source code is published in this repository. See [IP-PROTECTION-NOTICE.md](../IP-PROTECTION-NOTICE.md).
